Introduction

Cloth simulation has traditionally been done using many different methods. One of the most efficient methods used in modern graphics programming is the utilization of GPU side compute shaders to calculate the physical interactions for a simulated cloth plane. Compute shaders can be implemented with every industry standard modern graphics library, and for this project we will be adapting the work of Va Hongly, Min-Hyung Choi, and Min Hong in Vulkan.

Cloth simulation is important for modeling garments in media, like films, video games, contributing to both realism and stylistic rendering. Modeling this behavior contributes to scene aesthetics, immersion, and believable physics interactions. It is implemented using “the mass-spring system,” and both implicit and adaptive constraints [6]. Our project will implement these physics mechanisms using compute shaders to construct a believable cloth simulation.

Description and Methods

Overview

This project takes a plane and texture, applies according transforms, and runs through a shader pipeline to simulate cloth motion.

Process and Algorithm

  1. Development Environment
    • Download and install VulkanSDK
    • Configure Visual Studio and linker settings.
  2. Install and Link Additional Dependencies
    • GLM
    • GLFW
    • stb_image.h
    • tinyObjLoader
  3. Set Up Error Checking and Check Hardware
    • Configure validation layers for error handling.
    • Check for supported extensions.
    • Pick graphics card to use.
  4. Hello Triangle
    • Create graphics pipeline and pipeline setup.
    • Involves configuration of swapchain, imageviews, framebuffers, vertexbuffers and vertex and fragment shader modules.
  5. Next Steps
    • Implement MVP support for matrix transforms.
    • Implement depth buffering.
    • Load in meshes with tinyObjLoader.
  6. Compute Shader
    • Configure compute shader to pass in vertex positions and velocities.
    • Pass in UBO.
    • Implement mass-spring system.

Goals

  • Implement a forward renderer in Vulkan.
  • Using compute shaders, implement a spring-based cloth simulation.
  • Implement cloth interactions with simple objects.

Conclusion and Future Work

The implemented cloth simulation successfully adapts the previous work in OpenGL to Vulkan. Using compute shaders and the basic mechanics of the mass-spring system, we were able to complete a prototype cloth simulation. Through this project, we demonstrated significant growth in our understanding of Vulkan, GPU state management, and compute shaders.

There are several avenues for future work:

  • Refactor Vulkan code for scalability
  • Add obj interactivity
  • GUI for interaction
